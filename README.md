## Engineering Principles

These are the ideas that shape how I approach software development.

- Build for maintainability over shortcuts.
- Security is a feature, not an afterthought.
- Architecture should simplify change, not create unnecessary complexity.
- Learn the "why" behind technologies, not just the syntax.
- Solve real problems before reaching for new tools or frameworks.
- Keep learning. Software never stands still, and neither should I.
